# Conjure -- MVP Plan

**Conjure** is a macOS window switcher built with Rust + Tauri 2.

## Context

macOS default window switching (Cmd+Tab, Cmd+`) is slow when juggling multiple windows of the same app (e.g., several Cursor or Chrome windows). Conjure provides a keyboard-driven command palette with fuzzy search to switch between any open window instantly -- similar to Contexts or Raycast window search.

This plan covers the MVP: fast fuzzy window switching with no extras.

---

## Technology Choices

| Layer | Choice | Rationale |
|-------|--------|-----------|
| Framework | Tauri 2 | Native performance, small binary, Rust backend |
| Frontend | Vanilla TypeScript | Single view (input + list), no framework needed |
| Fuzzy search | `nucleo-matcher` | Battle-tested (helix-editor), fast, simple API |
| macOS APIs | `core-graphics` + `accessibility-sys` + `cocoa`/`objc` | CGWindowList for enumeration, AXUIElement for titles/activation |
| Global hotkey | `tauri-plugin-global-shortcut` | Official Tauri 2 plugin |
| System tray | `tauri::tray::TrayIconBuilder` | Built into Tauri 2 core, no extra plugin |

## Implementation Phases

### Phase 1: Scaffolding

1. Create Tauri 2 project with vanilla-ts template
2. Configure `Cargo.toml` dependencies:
   - `tauri` (with `macos-private-api` feature), `tauri-plugin-global-shortcut`
   - `nucleo-matcher`, `serde`, `serde_json`
   - `core-graphics`, `core-foundation`, `accessibility-sys`, `cocoa`, `objc`
3. Configure `tauri.conf.json` window: `decorations: false`, `transparent: true`, `alwaysOnTop: true`, `visible: false`, `center: true`, `resizable: false`, `skipTaskbar: true`, dimensions 680x420
4. Configure capabilities for global-shortcut permission
5. Create `Makefile` with standardized commands:
   - `make dev` -- run `cargo tauri dev`
   - `make build` -- run `cargo tauri build`
   - `make clean` -- remove build artifacts (`src-tauri/target`, `node_modules`, `dist`)
   - `make install` -- install all dependencies (`npm install`)
   - `make lint` -- run linters (clippy + tsc --noEmit)
   - `make fmt` -- format code (cargo fmt + prettier)
6. **Verify**: `make dev` shows empty transparent floating window

### Phase 2: App Icon & Menu Bar Tray

Files: `src-tauri/icons/`, `src-tauri/src/lib.rs`

1. Design app icon: a minimal wand/sparkle glyph -- represents "conjuring" windows
   - Simple geometric shape that reads well at 16px and 1024px
   - Monochrome base so the tray template image works with macOS light/dark mode
   - Generate sizes: 1024x1024 PNG, .icns for app bundle, 22x22 + 44x44 template PNGs for tray
2. Configure `tauri.conf.json`: set `icon` paths, set `"activationPolicy": "accessory"` to hide from Dock (background-only app)
3. Set up system tray via `TrayIconBuilder` in the Tauri setup block:
   - Use template tray icon (monochrome, macOS auto-adapts to light/dark)
   - Menu items: "About Conjure", separator, "Quit" (with `Cmd+Q` accelerator)
4. **Verify**: App shows icon in menu bar, no Dock icon, tray menu works

### Phase 3: Accessibility Permissions

File: `src-tauri/src/permissions.rs`

1. Check `AXIsProcessTrusted()` on startup
2. If not granted, call `AXIsProcessTrustedWithOptions` with prompt flag to show system dialog
3. Emit permission status to frontend so it can show a notice if needed
4. **Verify**: App prompts for accessibility on first launch

### Phase 4: Window Enumeration

Files: `src-tauri/src/windows/enumerate.rs`, `types.rs`, `mod.rs`

1. Call `CGWindowListCopyWindowInfo` to get all windows
2. Filter out: layer != 0, empty owner names, system processes (Window Server, Dock), tiny windows (<50px), own PID
3. For each unique PID, use `AXUIElementCreateApplication` + `kAXWindowsAttribute` to get window titles
4. Correlate AX windows to CG windows by position/size matching
5. Build `WindowInfo` structs with: `id`, `app_name`, `app_pid`, `title`, `app_bundle_id`, `is_minimized`, `display_text`
6. **Verify**: Tauri command returns JSON list of open windows in dev console

### Phase 5: Frontend v1

Files: `src/index.html`, `src/main.ts`, `src/style.css`

1. HTML: `#palette` div with search `<input>` and `<ul id="results">`
2. CSS: transparent body, dark palette with `backdrop-filter: blur(20px)`, `border-radius: 12px`, keyboard-driven selection highlight (no mouse hover states), `overflow-y: auto`, `cursor: none` on palette
3. TypeScript:
   - On palette open: call `search_windows("")`, render MRU-ranked list, focus input
   - On input change: call `search_windows(query)`, re-render, reset selection to 0
   - ArrowUp/ArrowDown (or Tab/Shift+Tab): move selection, scroll into view
   - Enter: call `activate_window`, hide palette
   - Escape: hide palette
   - Dismiss palette on window focus loss (e.g., user Cmd+Tabs away)
4. **Verify**: Palette shows list of windows on open

### Phase 6: Window Activation

File: `src-tauri/src/windows/activate.rs`

1. Find `NSRunningApplication` for target PID via `runningApplicationWithProcessIdentifier:`
2. Call `activateWithOptions: .activateIgnoringOtherApps`
3. Find matching AXUIElement window for the target
4. If minimized, set `kAXMinimizedAttribute` to false
5. Perform `kAXRaiseAction` to bring exact window to front
6. macOS auto-switches Spaces when raising a window via AX API
7. **Verify**: Press Enter on a selected window, it switches to that window

### Phase 7: Global Hotkey

File: `src-tauri/src/lib.rs` (setup block)

1. Register `Cmd+Ctrl+Space` via `tauri-plugin-global-shortcut`
2. On press: toggle palette visibility (show + center + focus, or hide)
3. Emit `palette-opened` event to frontend on show
4. **Verify**: Hotkey shows/hides palette from any app

### Phase 8: Fuzzy Search

File: `src-tauri/src/search/mod.rs`

1. Use `nucleo-matcher` with `Pattern::parse` (case-insensitive, smart normalization)
2. Match against `display_text` ("AppName - Window Title")
3. Return results sorted by score descending
4. Empty query returns all windows in MRU order
5. **Verify**: Type in input, list filters in real-time

### Phase 9: MRU Tracking

File: `src-tauri/src/mru/mod.rs`

1. In-memory ring buffer of `(bundle_id, title)` pairs, most recent first
2. On `activate_window`: record activation, move to front
3. Apply MRU ranking before returning search results
4. Session-only (no persistence for MVP)
5. **Verify**: Recently activated windows appear first in empty-query results

### Phase 10: Documentation

1. Write `README.md`: project description, features, prerequisites (Rust, Node), build/run instructions, permissions setup, hotkey info
2. Write `CLAUDE.md`: project-specific instructions for Claude Code -- architecture overview, module responsibilities, build commands, testing approach, code conventions

### Phase 11: Polish

1. Focus management: ensure input always has focus when palette opens
2. Edge cases: handle windows that close between enumeration and activation
3. Hide palette after activation
4. Clear input on palette open
5. **Verify**: Full workflow -- hotkey -> type -> enter -> window switches

## Key macOS API Details

**Window enumeration** (CGWindowListCopyWindowInfo): Fast (~1-3ms), returns window IDs, owner names, PIDs, bounds. Often returns empty titles -- must enrich via AX API.

**Title enrichment** (AXUIElement): For each PID, get `kAXWindowsAttribute` array, read `kAXTitleAttribute` from each. Match to CG windows by position/size. Slower (~10-20ms total) but necessary.

**CGWindowID to AXUIElement correlation**: No direct API exists. Match by window bounds. Fallback: private `_AXUIElementGetWindow` function (used by alt-tab-macos).

**Window activation sequence**: `NSRunningApplication.activate` -> `AXRaise`. This handles cross-Space switching automatically.

## Performance Targets

| Metric | Target | Approach |
|--------|--------|----------|
| Palette open | < 30ms | Pre-configured hidden window, just show + center |
| Search update | < 10ms | nucleo-matcher on ~50-200 items, synchronous |
| Window activation | < 50ms | Direct AX API calls, no intermediary |
| Enumeration | < 30ms | CGWindowList + AX enrichment on palette open only |

---

## Known Risks

1. **CG-to-AX window matching**: Position-based correlation covers ~95% of cases. Fallback to `_AXUIElementGetWindow` private API if needed.
2. **Transparent window rendering**: Tauri + macOS can have issues. Mitigation: `macOSPrivateApi: true`. Fallback: opaque dark window.
3. **Cmd+Ctrl+Space conflicts**: Low risk -- uncommon shortcut. Configurable hotkey deferred to post-MVP.
4. **`accessibility-sys` completeness**: May need raw `objc` FFI for missing bindings.

## NOT in MVP (Deferred)

- App icons in results
- Persistent MRU across sessions
- Configurable hotkey
- Window preview thumbnails
- Multi-monitor awareness
- Workspace/Space filtering
- Auto-updater / DMG packaging / code signing
- Clipboard history integration
- Window pinning
